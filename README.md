## KiwiVM 是什么？搬瓦工的自研控制面板简介

KiwiVM，全称 KiwiVM Control Panel，是搬瓦工专门为管理旗下 VPS 产品而开发的控制面板系统。每购买一台搬瓦工 VPS，系统就会为它生成一个独立的 KiwiVM 面板实例，里面包含这台机器的所有核心信息：IP、SSH 端口、流量用量、CPU 和内存使用情况，以及一系列操作入口。

底层虚拟化用的是 KVM，不是老式的 OpenVZ。这意味着每台 VPS 有独立的内核空间，资源隔离做得更好，跑 Docker 和 tun/tap 类的应用也没什么障碍。

面板界面是英文的，但功能分区很清晰，上手不难。左侧菜单分三大块：**Admin functions**（日常管理）、**Migration**（机房迁移）、**KiwiVM Extras**（快照、备份等）。

## KiwiVM 全功能详解

### Admin Functions：日常管理

**Main Controls（主控界面）**

进面板第一眼看到的地方。右侧显示：

- 当前机器状态（运行中 / 已关机）
- VPS IP 地址和 SSH 端口
- 物理位置（当前机房）
- RAM、SWAP、磁盘使用情况
- 本月流量使用和重置日期
- 当前操作系统

左侧的操作按钮：**Start**（开机）、**Stop**（关机）、**Restart**（重启）。

这里还能看到一个细节——如果流量当月超出，面板会提示，并允许你临时恢复 VPS 访问权限，不用非得升级套餐才能进去处理紧急情况。

**Install New OS（重装系统）**

这个功能用起来不复杂：选择目标系统，点确认，等几分钟。支持 Ubuntu、Debian、CentOS、AlmaLinux、Rocky Linux、Fedora 等主流 Linux 发行版，32 位 64 位都有，最近还加上了 Ubuntu 26.04 和 Debian 13。

重装后 root 密码会发到注册邮箱，或者直接在下面的密码修改功能里重置。

**Root Password Modification（修改 root 密码）**

忘了密码或者想换一个更安全的，在这里改。操作完成后会发确认邮件。

**Two-Factor Authentication（两步验证）**

给 KiwiVM 面板本身加个两步验证。配合 Google Authenticator 或 Authy 使用，扫码绑定后登录时需要额外输入 6 位动态码。习惯用这类安全工具的朋友建议开启，毕竟面板里能重装系统、改 root 密码，权限不小。

**Private Network（私有内网）**

同一机房下的搬瓦工 VPS 实例之间，可以通过内网互联，传输数据不计入公网流量。有多台机器、需要内部数据传输的场景用得上，比如主从数据库同步、内网代理转发等。

**Detailed Statistics（详细统计）**

展示这台 VPS 的历史使用情况，包括：
- 带宽 I/O 曲线
- CPU 使用率图表
- 硬盘 I/O 数据

排查性能问题的时候会用到，平时看着玩也挺直观。

**Root Shell Basic / Advanced（在线 SSH）**

不用本地 SSH 客户端，直接在浏览器里连接到 VPS 命令行。Basic 版本基本够用，Advanced 版本支持复制粘贴操作，体验好一点。

有时候 SSH 客户端连不上（比如端口被封了），这个网页版终端就是救命工具。

**Audit Log（操作日志）**

记录这台 VPS 在 KiwiVM 里的所有操作历史：开关机时间、API 调用、密码修改……有安全意识的朋友可以定期看看，发现异常登录或操作比较方便。

### Migration：机房迁移

**Migrate to Another DC（迁移机房）**

这是搬瓦工 KiwiVM 里最有特色的功能，没有之一。

支持迁移的套餐（主要是 CN2 GIA-E 套餐和部分其他套餐），可以在超过 10 个机房之间自由切换，操作步骤如下：

1. 进入 KiwiVM，点左侧 **Migrate to another DC**
2. 在机房列表里选目标机房
3. 点 **Confirm on next step**
4. 再点 **Start Migration**
5. 等待 10-15 分钟，收到邮件通知后，用新 IP 重新连接

换机房后 IPv4 会变，这个要注意。如果用了域名，解析要更新。

还有一点要留意：迁移过程会消耗流量（相当于把整个磁盘数据传一次），迁移前可以清理一下不必要的大文件，省点月流量。

### KiwiVM Extras：快照与备份

**Snapshots（快照）**

手动给 VPS 创建一个完整备份，包含所有数据和系统配置。升级系统、改重要配置之前建一个快照，出问题了点一下就恢复，不用担心整个环境垮掉。

快照是免费的，这在 VPS 行业里不是理所当然的——不少商家这个功能要额外收费，搬瓦工直接包进去了。

**Backups（自动备份）**

区别于手动快照，这个是系统定期自动备份，周期由系统根据你的数据量自动计算。你不用设置任何东西，备份就在后台跑着。需要恢复的时候，把备份导入快照，再从快照还原即可。

## KiwiVM API：自动化管理 VPS

对有开发需求的朋友来说，KiwiVM 提供了一套 API 接口。在面板左侧找到 **API**，获取你的 **API KEY** 和 **VEID**，就可以用 Python、PHP、Shell 脚本等调用 API，实现自动化管理——查看 VPS 状态、流量用量、触发重启、读取实时统计数据等。

不需要写代码的朋友忽略这块就好。

## 搬瓦工 KiwiVM 使用过程中常见问题

**Q：KiwiVM 面板登录不进去怎么办？**

先确认账号密码正确。如果通过客户区跳转进入，注意要保持登录状态。如果面板提示 "Access Denied"，等几分钟重试，可能是 session 超时。

**Q：迁移机房之后 IP 被封了，可以再迁回来吗？**

不可以直接迁。IP 被封的状态下，KiwiVM 里的迁移机房功能会灰掉。这个时候有两个选择：花钱换 IP（搬瓦工提供付费换 IP 服务），或者联系工单说明情况。

**Q：KiwiVM 支持安装 Windows 系统吗？**

官方系统模板里有 Windows 10 的挂载选项，但只是挂载 ISO，需要手动完成安装流程，步骤比 Linux 复杂不少。如果你的需求场景是 Windows，可以试，但做好心理准备。

**Q：搬瓦工流量超了会怎样？**

VPS 会被暂停，但 KiwiVM 里有个功能可以临时恢复——不需要升级套餐，临时恢复后能进去处理紧急数据。流量月底重置，重置后自动恢复正常。
